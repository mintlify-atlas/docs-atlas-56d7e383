## Quick Start with Docker Hub

The fastest way to run the app is using the pre-built image from Docker Hub:

<Steps>
  <Step title="Pull the image">
    Download the official image:
    
    ```bash
    docker pull thisisrober/aws-exam-prep-app:latest
    ```
    
    Expected output:
    ```plaintext
    latest: Pulling from thisisrober/aws-exam-prep-app
    Digest: sha256:abc123...
    Status: Downloaded newer image for thisisrober/aws-exam-prep-app:latest
    ```
  </Step>
  
  <Step title="Run the container">
    Start the container:
    
    ```bash
    docker run -d \
      --name aws-exam-app \
      -p 5173:5173 \
      --restart unless-stopped \
      thisisrober/aws-exam-prep-app:latest
    ```
    
    <Info>
    The `-d` flag runs the container in detached mode (background)
    </Info>
  </Step>
  
  <Step title="Access the application">
    Open your browser and navigate to:
    
    **http://localhost:5173**
    
    <Success>
    The application is now running in a Docker container!
    </Success>
  </Step>
</Steps>

## Building from Source

If you want to customize the image or build from source code:

### Dockerfile Overview

The application uses a multi-stage build for optimal image size:

```dockerfile Dockerfile
# Stage 1: Build
FROM node:18-alpine AS builder

WORKDIR /app

# Copy package files
COPY package*.json ./

# Install dependencies
RUN npm ci

# Copy application files
COPY . .

# Build the application
RUN npm run build

# Stage 2: Runtime
FROM node:18-alpine

WORKDIR /app

# Install serve to host the built app
RUN npm install -g serve

# Copy built application from builder stage
COPY --from=builder /app/dist ./dist

# Create non-root user for security
RUN addgroup -g 1001 -S nodejs && \
    adduser -S nodejs -u 1001
USER nodejs

# Expose port
EXPOSE 5173

# Health check
HEALTHCHECK --interval=30s --timeout=10s --start-period=5s --retries=3 \
  CMD node -e "require('http').get('http://localhost:5173', (r) => {if (r.statusCode !== 200) throw new Error(r.statusCode)})"

# Start the application
CMD ["serve", "-s", "dist", "-l", "5173"]
```

### Build Your Own Image

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/thisisrober/aws-certified-app.git
    cd aws-certified-app
    ```
  </Step>
  
  <Step title="Build the Docker image">
    ```bash
    docker build -t aws-exam-app:custom .
    ```
    
    The build process:
    1. Installs dependencies with `npm ci`
    2. Builds the production bundle with `npm run build`
    3. Creates a lightweight runtime image
    4. Copies only the built files
    
    <Info>
    The build typically takes 2-5 minutes depending on your system.
    </Info>
  </Step>
  
  <Step title="Run your custom image">
    ```bash
    docker run -d \
      --name aws-exam-app \
      -p 5173:5173 \
      aws-exam-app:custom
    ```
  </Step>
</Steps>

## Docker Compose Deployment

For easier management and configuration, use Docker Compose:

### docker-compose.yml

The project includes a `docker-compose.yml` file:

```yaml docker-compose.yml
version: '3.8'

services:
  app:
    build:
      context: .
      dockerfile: Dockerfile
    container_name: aws-exam-app
    ports:
      - "5173:5173"
    environment:
      - NODE_ENV=production
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "node", "-e", "require('http').get('http://localhost:5173', (r) => {if (r.statusCode !== 200) throw new Error(r.statusCode)})"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 5s
    networks:
      - app-network

networks:
  app-network:
    driver: bridge
```

docker logs aws-exam-app

docker logs -f aws-exam-app

docker logs --tail 100 aws-exam-app

# Check container status
docker ps

## Configuration Options

### Port Mapping

Change the host port while keeping container port 5173:

```bash
# Run on port 8080
docker run -d -p 8080:5173 --name aws-exam-app thisisrober/aws-exam-prep-app:latest

# Access at http://localhost:8080
```

### Environment Variables

Pass environment variables to the container:

```bash
docker run -d \
  --name aws-exam-app \
  -p 5173:5173 \
  -e NODE_ENV=production \
  thisisrober/aws-exam-prep-app:latest
```

### Resource Limits

Limit CPU and memory usage:

```bash
docker run -d \
  --name aws-exam-app \
  -p 5173:5173 \
  --memory="512m" \
  --cpus="0.5" \
  thisisrober/aws-exam-prep-app:latest
```

### Restart Policies

<Tabs>
  <Tab title="unless-stopped">
    Restart unless manually stopped:
    ```bash
    docker run -d --restart unless-stopped aws-exam-app
    ```
  </Tab>
  
  <Tab title="always">
    Always restart (even after reboot):
    ```bash
    docker run -d --restart always aws-exam-app
    ```
  </Tab>
  
  <Tab title="on-failure">
    Restart only on failures:
    ```bash
    docker run -d --restart on-failure:3 aws-exam-app
    ```
  </Tab>
</Tabs>

## Health Checks

The Docker image includes a built-in health check:

```bash
# Check health status
docker inspect --format='{{json .State.Health}}' aws-exam-app | jq
```

Health check configuration:
- **Interval**: 30 seconds
- **Timeout**: 10 seconds
- **Start Period**: 5 seconds
- **Retries**: 3 attempts

<Info>
The health check makes an HTTP request to `http://localhost:5173` and verifies a 200 status code.
</Info>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Container exits immediately">
    Check the logs for errors:
    
    ```bash
    docker logs aws-exam-app
    ```
    
    Common causes:
    - Build errors in the application
    - Port already in use
    - Insufficient permissions
  </Accordion>
  
  <Accordion title="Port binding error">
    If you see "port is already allocated":
    
    ```bash
    # Find what's using port 5173
    lsof -i :5173  # macOS/Linux
    netstat -ano | findstr :5173  # Windows
    
    # Use a different port
    docker run -d -p 8080:5173 --name aws-exam-app thisisrober/aws-exam-prep-app:latest
    ```
  </Accordion>
  
  <Accordion title="Cannot connect to Docker daemon">
    Ensure Docker is running:
    
    ```bash
    # Check Docker status
    docker info
    
    # Start Docker Desktop (if installed)
    # Or start Docker service
    sudo systemctl start docker  # Linux
    ```
  </Accordion>
  
  <Accordion title="Image pull fails">
    If pulling from Docker Hub fails:
    
    1. Check your internet connection
    2. Verify the image name is correct
    3. Try with explicit tag:
    
    ```bash
    docker pull thisisrober/aws-exam-prep-app:latest
    ```
    
    4. If behind a proxy, configure Docker:
    
    ```bash
    # Create/edit ~/.docker/config.json
    {
      "proxies": {
        "default": {
          "httpProxy": "http://proxy:port",
          "httpsProxy": "http://proxy:port"
        }
      }
    }
    ```
  </Accordion>
  
  <Accordion title="Build fails with memory error">
    Increase Docker memory limit:
    
    **Docker Desktop**:
    1. Open Settings → Resources
    2. Increase Memory to at least 2GB
    3. Click Apply & Restart
    
    **Linux**:
    ```bash
    # Build with resource limits
    docker build --memory=2g -t aws-exam-app .
    ```
  </Accordion>
</AccordionGroup>
